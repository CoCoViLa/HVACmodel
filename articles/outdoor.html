Component: Outdoor conditions

The main idea of the OutdoorConditions component is to simulate the properties and behavior of the air and outdoor conditions. There are several constant parameters, such as heat capacity or air density, also there is one time dependent parameter - temperature, so the temperature is different for day and night periods.

The component has several properties:

  • name
  • airDeinsity (default value 1.225) - density of air at sea level 1.225 (kg/m^3);
  • c (default value 1005.4) - heat capacity of air (273K) 1005.4 (J/kg-K);
  • min (default value -10.0) - minimal temperature (C);
  • max (default value 30.0) - maximal temperature (C);

The outputs of the component are air density, heat capacity and temperature, the input is signal from timer.

The most complex part here is modeling temperature daily variations. Initially, we have two input parameters min and max. We constructed the time-temperature function using sinusoid so, that the maximal value of the temperature is at 14.00.

The simulation results for the scheme outdoor.syn.

The outdoor temperature outdoorT changes over the time according timer signal. The temperature daily variation function can be redefined if required.
