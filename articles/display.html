Component: Display

This component is required for setting up the temperature we want to hold indoors. There are two possibilities: the easiest way is set up the constant temperature for the whole day, this is modeled by Display, the component DisplayDN allows to model more complex situation, where we can assign two temperatures for day and night.

Display

The Display is required for setting up the required temperature we want to get indoors. The temperature value is user defined parameter and stays constant for the whole time of simulation.

There are only two parameters for this component: name and the temperature value. The default value fortemperature is 21 (C). So, the output of this component is temperature.

To illustrate the work-flow of this component we can compose the Scheme (also can be found in display_simple.syn), that visualizes the behavior of the component during period of time .

As you can see, the output value temperature is constant.

DisplayDN

Compared to previous component Display, the DisplayDN has more complex behavior. It is done to present the situation, when we need to set up two different temperature values for day and night.

The properties of the component are the following :

  • name;
  • temperatureDay (default value 21). The temperature value for the day (C);
  • temperatureNight (default value 18). The temperature value for the night (C);
  • nightBegin (default value 22.00). Time, when night begins;
  • nightEnd (default value 6.00). Time, when night ends;

The output of the component is temperature. The input of the component is time signal from timer.

For simulation of the behavior of the DisplayDN component we composed scheme display.syn .

As you can see the output depends on time according to presented parameters.
